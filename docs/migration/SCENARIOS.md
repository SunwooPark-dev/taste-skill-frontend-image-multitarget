# SCENARIOS

아래는 지금 이 저장소를 3개 환경에서 **바로 써볼 수 있는 시나리오**입니다.

---

## Scenario A — Hero 최소 개선

### 목적
과한 리디자인 없이 hero만 더 premium하게 만든다.

### Codex App
```text
이 저장소의 adapters/codex/AGENTS.md 기준으로 작업해.
기존 landing hero를 더 프리미엄하게 바꿔줘.
다른 섹션은 건드리지 말고, 검증 기준부터 먼저 적어.
```

### Antigravity App
```text
기존 hero를 더 premium하게 만들고 싶어.
나머지는 크게 건드리지 말아줘.
먼저 어떤 가정을 하고 있는지 적고 시작해줘.
```

### Gemini terminal workflow
```text
adapters/gemini/GEMINI_PROMPT.md 기준으로 작업해.
hero만 개선해.
기존 구조는 유지하고, 변경 후 검증 방식까지 적어.
```

## Scenario B — Image-first가 필요한지 판단

### 목적
무조건 이미지를 넣지 말고, 실제로 image-first 접근이 필요한지 먼저 판단한다.

### Codex App
```text
adapters/codex/AGENTS.md 기준으로 판단해.
이 페이지가 image-first 접근이 필요한지 먼저 결정하고,
필요할 때만 구조적으로 반영해줘.
```

### Antigravity App
```text
이 페이지를 더 premium하게 만들고 싶어.
근데 이미지 중심 접근이 정말 필요한지부터 판단해줘.
모호하면 해석 옵션도 같이 제시해줘.
```

### Gemini terminal workflow
```text
adapters/gemini/GEMINI_PROMPT.md 기준으로 작업해.
image-first가 필요한지 먼저 판단하고,
불확실하면 가정을 적고 보수적으로 진행해.
```

## Scenario C — Generic AI 느낌 제거

### 목적
흔한 AI SaaS 스타일을 줄이고 더 구조적인 premium 방향으로 정리한다.

### Codex App
```text
이 저장소의 codex adapter 기준으로 작업해.
현재 hero가 너무 generic AI product처럼 보여.
구조는 유지하되 hierarchy, spacing, CTA 우선순위만 더 좋게 바꿔줘.
```

### Antigravity App
```text
현재 페이지가 너무 generic AI product처럼 보여.
최소 변경으로 premium하게 만드는 방향을 제안해줘.
```

### Gemini terminal workflow
```text
adapters/gemini/GEMINI_PROMPT.md 기준으로 작업해.
generic AI 느낌만 줄이고 싶어.
기존 구조를 유지한 채 수정 포인트와 검증 기준을 적어줘.
```
