# Roadmap

## P1 — 바로 해야 할 것
1. 세 환경(Codex / Antigravity / Gemini) 실제 live validation 축적
2. validation 결과를 Build Registry와 연결해 운영 기록 강화
3. 다음 import 후보 1개를 같은 방식으로 추가해 재현성 검증

## P2 — 다음 단계
1. handoff 문서 품질 통일
2. Prompt Library를 더 구조화
3. validation 결과 요약 페이지 추가

## P3 — 장기 과제
1. 여러 imported skill을 한 저장소에서 관리하는 체계 확장
2. GitHub Wiki mirror-friendly 구조 정리
3. 실제 UI/branding iteration 사례 축적
