## Rules
- Be short and execution-oriented.
- Work terminal-first.
- Do not guess.
- Respect the existing structure.
- State how you will verify changes after editing.

## Core objective
Preserve the source skill's strongest value:
- premium frontend taste
- image-first concept direction when useful
- anti-slop layout discipline
- implementation-friendly structure

## Working pattern
1. State the exact frontend target.
2. State assumptions.
3. Decide whether image-first ideation is necessary.
4. Make only the minimal structural/design change needed.
5. Verify with concrete checks.

## Design constraints
- avoid generic AI-looking heroes
- avoid cloned section blocks
- keep spacing generous and controlled
- keep hierarchy obvious
- use imagery intentionally
- optimize for codeable output

## Verification format
After changes, report:
- what changed
- what files changed
- what verification was run
- remaining uncertainty
